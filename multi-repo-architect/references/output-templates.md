## 🔴 Critical Issues

### [CRITICAL] Hardcoded AWS credentials in application code
- **Category**: Security
- **Affected**: `payments-service` — `src/config/aws.ts:12`
- **Description**: AWS Access Key and Secret are hardcoded in source code, committed to the repository. Anyone with repo access can exfiltrate data from the AWS account.
- **Evidence**: `AWS_ACCESS_KEY_ID = "AKIA..."` in `payments-service/src/config/aws.ts`
- **Recommendation**: Remove immediately, rotate the credentials, use IAM roles for EC2/ECS or environment injection via AWS Secrets Manager.

---

## 🟠 High Issues

### [HIGH] RDS instance publicly accessible
- **Category**: Security
- **Affected**: `infrastructure` — `terraform/rds.tf`
- **Description**: The production RDS instance has `publicly_accessible = true`, exposing the database endpoint to the internet.
- **Evidence**: `resource "aws_db_instance" "main" { publicly_accessible = true }`
- **Recommendation**: Set `publicly_accessible = false`. Place RDS in a private subnet and access via application servers in the same VPC.

---

## 🟡 Medium Issues

### [MEDIUM] No health checks on critical services
- **Category**: Reliability
- **Affected**: `docker-compose.yml` — `payments-service`, `inventory-service`
- **Description**: These services have no `healthcheck` defined in docker-compose, meaning Docker cannot detect and restart failed containers automatically.
- **Evidence**: Neither `payments-service` nor `inventory-service` blocks contain a `healthcheck` key.
- **Recommendation**: Add healthcheck to each service:
  ```yaml
  healthcheck:
    test: ["CMD", "curl", "-f", "http://localhost:8080/health"]
    interval: 30s
    timeout: 10s
    retries: 3
  ```

---

## 🔵 Low Issues

### [LOW] Docker images use :latest tag
- **Category**: Reliability
- **Affected**: `docker-compose.yml` — `redis`, `postgres`
- **Description**: Using `:latest` tags for dependency images means updates can silently introduce breaking changes.
- **Evidence**: `image: redis:latest`, `image: postgres:latest`
- **Recommendation**: Pin to specific versions: `redis:7.2-alpine`, `postgres:15.4`
